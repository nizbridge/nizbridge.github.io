# F1 레이싱 게임

2026 F1 그리드 기반 레이싱 시뮬레이션. 타이어 전략과 ERS 배터리 관리가 핵심.

---

## 게임 흐름

1. **서킷 선택** — 모나코 / 실버스톤 / 몬자
2. **드라이버 선택** — 2026 F1 22명
3. **타이어 선택** — 소프트 / 미디엄 / 하드
4. **레이스** — 10랩, 카운트다운 후 스타트
5. **결과** — 포디움 + 전체 랩타임 테이블

---

## 속도 계산

모든 차의 속도는 매 프레임 아래 공식으로 결정된다.

```
최종 속도 = base_speed × cornerFactor × tireFactor × batFactor
```

| 요소 | 설명 |
|------|------|
| `base_speed` | 드라이버 레이팅 기반. `pLen / (20 - rating/100×5 + random±0.8)`. 매 랩마다 재계산 |
| `cornerFactor` | 트랙 곡률 기반 코너 감속 (0.28 ~ 1.0). 헤어핀 ≈ 0.28, 직선 ≈ 1.0 |
| `tireFactor` | 타이어 마모도에 따른 배수 |
| `batFactor` | 배터리 잔량 및 ERS 모드에 따른 배수 |

### 코너 속도 (cornerFactor)

트랙 경로를 1000개 샘플로 분할 후 각 지점의 곡률을 분석한다.
- W=25 샘플 앞뒤 벡터의 cos 각도 → 속도 배수로 변환
- 브레이킹 존: 60샘플 전방 예측으로 코너 전 미리 감속

## 타이어 시스템

### 컴파운드별 성능

| 컴파운드 | 신선 속도 | 완전 마모 속도 | 마모 속도 | 교체 권장 |
|---------|---------|------------|---------|---------|
| **S (소프트)** | ×1.06 | ×0.88 | 30%/랩 | ~3랩 |
| **M (미디엄)** | ×1.00 | ×0.92 | 17%/랩 | ~6랩 |
| **H (하드)**   | ×0.96 | ×0.91 | 10%/랩 | ~10랩 |

### 마모도 계산

```
tireFactor = freshSpd - (tireWear/100) × (freshSpd - wornSpd)
```

- 마모도 0%: 신선 속도 적용
- 마모도 100%: 완전 마모 속도 적용
- 마모 속도는 주행 거리에 비례 (코너/직선 구분 없음)

### 타이어 상태 표시 (하단 패널)

| 색상 | 마모도 |
|------|-------|
| 청록색 | 0 ~ 49% |
| 노란색 | 50 ~ 79% |
| 빨간색 | 80 ~ 100% |

### 피트 스톱

- **소요 시간**: 3~6초 랜덤 (플레이어/AI 동일)
- **효과**: 타이어 마모 0% 리셋, 선택한 컴파운드로 교체 (배터리는 유지)
- **공통**: 스타트/피니시 라인 통과 시 피트인 실행 (플레이어·AI 모두)
- **AI 피트 기준**: 마모도 88% 초과 + 남은 랩 1랩 이상 → 다음 스타트 라인에서 자동 피트
- **플레이어 피트 흐름**:
  1. 하단 `PIT IN` 버튼 클릭
  2. 타이어 컴파운드 선택 (S / M / H)
  3. 스타트/피니시 라인 통과 시 자동 피트인
  4. 버튼이 `PIT: X QUEUED` 상태로 바뀜 (취소 가능)

## ERS 배터리 시스템

### 기본 규칙

- 배터리는 0 ~ 100% (Overtake Mode 시 최대 150%)
- 코너에서 자동 충전 (회생 제동)
- 직선에서 ERS 방전 (노말/부스트 모드)
- **배터리 0%**: 속도 ×0.68로 급감

### 충전 / 방전 속도

| 상황 | 충전/방전 |
|------|---------|
| 직선 — 노말 모드 | −2.5%/s |
| 직선 — 부스트 모드 | −6.5%/s |
| 직선 — 리차지 모드 | +5.4%/s |
| 코너 — 노말 모드 | +5.0%/s |
| 코너 — 부스트 모드 | 0 (충전/방전 없음) |
| 코너 — 리차지 모드 | +23.2%/s |

---

## ERS 모드

하단 패널 버튼으로 전환. **같은 버튼 다시 누르면 노말로 복귀.**

### BOOST 모드 (주황색 활성)

- 속도 **×1.15** (+15%)
- 배터리 소모 **−6.5%/s** (직선)
- 언제든지 사용 가능, 배터리 충분할 때 추월 시 유리
- **배터리 0% 도달 시 자동으로 노말 모드로 전환**

### RECHARGE 모드 (파란색 활성)

- 속도 **×0.90** (−10%)
- 직선에서도 충전: +6.0%/s
- 코너에서 충전: +25.8%/s
- 배터리가 떨어졌을 때 빠르게 회복하는 용도

### NORMAL 모드 (기본)

- 속도 **×1.0** (배터리 있을 때)
- 직선 소모: −2.5%/s
- 코너 충전: +5.5%/s

### 배터리 0% 패널티

| 배터리 상태 | 속도 배수 |
|-----------|---------|
| > 0% | 모드별 정상 적용 |
| 0% | ×0.68 (−32%) |

## Overtake Mode

DRS와 유사하지만 배터리 기반. **자동 활성화**, 버튼 없음.

### 활성화 조건

스타트/피니시 라인 통과 시 **앞차와 0.1초 이내**이면 다음 랩 전체 활성화.

### 효과

| 항목 | 변화 |
|------|------|
| 배터리 최대치 | 100% → **150%** |
| 충전 속도 | **×1.3** (모든 충전 상황에 적용) |
| 속도 보너스 | 없음 (용량 증가 + 빠른 충전으로 더 오래 BOOST 사용 가능) |

### 비활성화

- 랩 통과 시 앞차와 1초 이상 벌어지면 다음 랩부터 해제
- 해제 시 배터리가 100% 초과분이면 100%로 강제 컷

### 표시

- 순위 옆 녹색 `OT` 배지 (해당 차량)
- 하단 패널 ERS 바 우측에 OT 전용 연장 게이지 + `OT` 뱃지

---

## AI 배터리 전략

| 조건 | AI 행동 |
|------|--------|
| 배터리 < 20% | 리차지 모드로 전환 |
| 배터리 ≥ 65% + 리차지 중 | 노말로 복귀 |
| 배터리 > 55% + 앞차 0.7초 이내 | 부스트 모드 시도 |
| 부스트 중 배터리 < 35% 또는 코너 | 노말로 복귀 |

```javascript
// 레이스
TOTAL_LAPS     = 10

// 속도 배수

RECH_MULT      = 0.90

// Overtake Mode
OT_REGEN_FACTOR= 1.3  // 충전 속도 배수
```
